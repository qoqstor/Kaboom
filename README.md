# Kaboom! — GitHub Pages 靜態版

這個資料夾是 **免後端** 的 Kahoot 風格互動問答網站，可直接放上 GitHub Pages（或任何靜態空間）運作。

## 運作原理

- 沒有伺服器：**主持人的瀏覽器就是遊戲伺服器**
- 玩家透過 WebRTC（PeerJS）點對點連到主持人，訊號交換使用 PeerJS 官方免費雲端伺服器
- 遊戲 PIN 就是 WebRTC 房間 ID，玩家輸入 PIN 即可配對
- 題庫儲存在瀏覽器 localStorage（僅存在建立者的裝置上），支援 JSON 匯出 / 匯入

## 部署到 GitHub Pages

1. 把這個資料夾的所有檔案上傳到 GitHub repo（放在根目錄，或保留在 `docs/` 資料夾）
2. Repo → **Settings → Pages** → Source 選 **Deploy from a branch**
3. Branch 選你的分支，資料夾選 `/ (root)` 或 `/docs`（依你上傳的位置）→ **Save**
4. 一兩分鐘後網站就會出現在 `https://你的帳號.github.io/repo名稱/`

## 頁面

| 頁面 | 用途 |
|---|---|
| `index.html` | 玩家：輸入 PIN + 暱稱作答 |
| `admin.html` | 題庫管理：建立 / 編輯 / 刪除 / 匯出匯入題庫、啟動遊戲 |
| `host.html` | 主持人大螢幕：顯示 PIN、題目、倒數、排行榜、頒獎台（由 admin 頁啟動） |

## 使用流程

1. 開 `admin.html` 建題庫（每題 2–4 選項、指定正確答案、作答 5–120 秒）
2. 按「🚀 開始遊戲」→ 大螢幕顯示 6 位數 PIN
3. 玩家開網站首頁輸入 PIN + 暱稱加入
4. 開始遊戲：限時搶答、答越快分越高、連對有加成、每題後排行榜、最後頒獎台

## 注意事項

- **主持人視窗整場要保持開啟**（它就是伺服器；關掉 = 遊戲結束）
- 題庫存在「建立它的那台裝置的瀏覽器」裡；換裝置請先匯出 JSON 再匯入
- 沒有登入機制：靜態網站無法安全保存密碼。其他人開你的網站也只會編輯到「他們自己瀏覽器裡」的題庫，動不到你的
- 連線人數建議 ≤ 100 人（P2P 架構，上限取決於主持人裝置與網路品質）
- 少數嚴格 NAT／防火牆環境（例如部分公司網路）可能無法建立 P2P 連線
- 若需要完整伺服器版（支援更多人、集中題庫、管理員登入），見 repo 根目錄的 Node.js 版本
